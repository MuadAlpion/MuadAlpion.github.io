# 💻 5. Responsive Design – ฉันออกแบบให้รองรับมือถืออย่างไร?

ตอบคำถามต่อไปนี้ (สั้น ๆ แต่มีเหตุผล)

### 5.1 หน้าจอมือถือ Persona อยากเห็นอะไร “เป็นอันดับแรก”?

_งานทั้งหมด และ Deadline ที่กำลังจะมาถึง_

### 5.2 ส่วนไหนถูกย้าย/ซ่อน/ย่อให้กระชับ?

- _เมนูแนวนอน → แนวตั้ง_  
- _ตาราง → แสดงเฉพาะช่องสำคัญ_  
- _การ์ด 3 ใบ → กลายเป็นเรียงลง_

### 5.3 Media Queries ที่ใช้ (อธิบายแนวคิดสั้น ๆ)

```css
@media (max-width: 768px) {
  /* ปรับ layout เป็นแนวตั้ง */
}

@media (max-width: 480px) {
  /* ลดฟอนต์/ลด spacing ให้พอดีกับมือถือ */
}
```

# 📝 8. Reflection – สิ่งที่ฉันได้เรียนรู้

ให้นศ.เลือกตอบ 3–5 ข้อ

- อะไรคือสิ่งที่ยากที่สุดใน LAB นี้?  
  → สิ่งที่ยากที่สุดคือการออกแบบหน้าเว็บให้รองรับการใช้งานบนมือถือเป็นหลัก (Mobile-first)ต้องคิดว่าเนื้อหาใดสำคัญที่สุดและควรถูกแสดงก่อน รวมถึงการย่อหรือซ่อนข้อมูลบางส่วนเพื่อให้หน้าเว็บไม่รกและยังใช้งานได้ง่ายบนหน้าจอขนาดเล็ก
- ฉันเรียนรู้อะไรเกี่ยวกับ UX ที่ไม่เคยคิดมาก่อน?  
  → ได้เรียนรู้ว่า UX ไม่ใช่แค่ความสวยงามของหน้าเว็บ แต่เป็นการออกแบบจากมุมมองของผู้ใช้จริง
- การใช้ Git/GitHub Pages ช่วยพัฒนางานอย่างไร?
  → การใช้ Git ช่วยให้สามารถจัดการเวอร์ชันของโค้ดและแก้ไขงานได้อย่างเป็นระบบ
  ส่วน GitHub Pages ทำให้สามารถเผยแพร่เว็บไซต์และทดสอบการใช้งานจริงบนอุปกรณ์ต่าง ๆ
  ช่วยให้เห็นปัญหาที่ไม่พบในระหว่างพัฒนาในเครื่องของตนเอง
